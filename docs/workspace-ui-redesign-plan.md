# レスポンシブ・ワークスペース UI 大改修計画

## 目的

現行の「ホーム + 多数のモーダル」構成を、ポータル本体の中で業務画面が切り替わるワークスペース UI へ段階移行する。

PC 全画面、PC 半分幅、スマホ幅で同じ機能を自然に扱えることを優先し、GitHub Pages + Supabase Free の前提は維持する。既存の `id` / `data-*` / Supabase データ構造は壊さず、必要な箇所から古いモーダル専用 CSS や未使用コードを削除する。

## 設計方針

- 画面遷移は本体内のビュー切り替えを基本にする。
- モーダルは「確認」「編集補助」「ピッカー」など短い作業だけに残す。
- 各業務画面は、まず既存 DOM をワークスペース内へ移動して壊さず動かし、その後で画面ごとに DOM / CSS を整理する。
- レスポンシブは 3 段階で見る。
  - PC 全画面: 左ナビ + 広い作業領域。
  - PC 半分幅: ナビを圧縮して、作業領域は 1 カラムへ自然に折り返す。
  - スマホ幅: 下部ナビ + 本体 1 カラム。横スクロールを出さない。
- ライト / ダークの両方で成立する CSS 変数中心の見た目にする。

## 改修順

1. ワークスペース基盤
   - 共通ヘッダー、戻る導線、active ナビ、レスポンシブ幅を整える。
   - 既存モーダルを本体内へ載せる仕組みを安定化する。

2. 部門間依頼
   - 依頼一覧、提出フォーム、目安箱をワークスペース画面として再構成する。
   - ステータス更新やタスク化など短い確認だけモーダルに残す。

3. タスク管理
   - 受信 / 送信 / 共有タスクをビュー内タブとして整理する。
   - 新規作成と編集フォームを同じ画面内に寄せる。

4. 勤怠カレンダー
   - カレンダー本体、日別入力、勤務内容表を 1 つの作業画面として整理する。
   - スマホでは日別入力を下に流す。

5. 発注
   - 発注入力、履歴、管理を作業画面内で切り替える。
   - プレビュー / 送信確認だけ短いモーダルに残す。

6. 共有リンク
   - 共有リンク一覧、カテゴリ、AI 提案を本体画面に移す。
   - 起動時に全件取得しない軽量設計は維持する。

7. メール / プロフィール
   - メール作成とプロフィール設定を用途別ビューに分ける。
   - 署名・連絡先編集は画面内パネル化する。

8. チャット / ファイル転送
   - パネルではなく本体ビューとして扱う。
   - DM / グループ / P2P / Drive をスマホでも行き来しやすくする。

9. 旧コード整理
   - 各機能移行後に、未使用 CSS、古いモーダル専用分岐、Firebase runtime 名残を削除する。
   - 機能追加と削除リファクタはコミットを分ける。

## 各機能の完了条件

- PC 1440px、PC 半分幅、スマホ 390px で横スクロールがない。
- ライト / ダークで読める。
- 主要導線が本体ビュー内で切り替わる。
- 保存、閉じる、再表示、関連画面反映が維持される。
- コンソール error / SEVERE が 0 件。
- 不要になった古い CSS / JS は同じ機能の仕上げコミットで削る。
